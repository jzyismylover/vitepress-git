vue 模版根节点 #

Vue2 模版里面只允许存在一个根节点，虽然Vue3 是允许多根节点的，但是建议还是保持单一个根节点的原则以免遇到不必要的警告。

<template>
  <div id="nav">
    <router-link to="/">全局分析</router-link>
    <router-link to="/about">热点聚集</router-link>
    <router-link to="/week">每周报表</router-link>
    <router-link to="/special">专题分析</router-link>
    <router-link to="/find">人工检索</router-link>
  </div>
  <router-view />
</template>

<template>
<!-- 保持单一根节点 -->

简单 mock #

简单使用 axios 和 mock 进行数据模拟。详见 WordCloudChart.vue

# 首先需要安装依赖
npm install axios
npm install mockjs

// 这种方式每个问题都要重新写一遍 -- 可以想想怎么样去优化(不是重点)
// 1. 可以将 mock 抽成 Vue 全局变量(不用每次都 import了)
// 2. 使用 webpack 或者 vite 的一些插件(自行探索)
import Mock from 'mockjs'
import axios from 'axios'
Mock.mock(/wordcloud/, 'get', {
  code: 200,
  'data|6': [{
    name: '@cname',
    value: '@integer'
  }]
})
axios.get('/wordcloud', {
  method: 'GET'
}).then(res => {
  console.log(res)
})

eslint 的配置 #

规则设置

  • off -- 关闭
  • warn -- 警告
  • error -- 报错

也就是说我们可以设置规则的不同级别，对于一些我们其实不那么习惯的规则可以把它关闭，也就是设置成 off .

// 其实 这里的配置是可以抽取到一个 .eslintrc.js 文件中去的
// 这样子 package.json 文件就不会显得很冗余
// 默认放在一起应该是在建项目的时候选择了将所有的东西都放到 package.json 这个选项了
"eslintConfig": {
    "root": true,
    "env": {
      "node": true
    },
    "extends": [
      "plugin:vue/vue3-essential",
      "eslint:recommended",
      "@vue/typescript/recommended",
      "@vue/prettier",
      "@vue/prettier/@typescript-eslint"
    ],
    "parserOptions": {
      "ecmaVersion": 2020
    },
    "rules": {
      // 对某些规则进行级别的限制如下
      // 这些规则其实都是 extends 里面
      "accessor-pairs":"off",
      "prefer-arrow-callback": "off"
    }
  },

  • 第二个爆红的地方是在 home.vue 每一行结束的时候，都是有一个 Delete 'CR' 这样子的一个东西，其实是 prettier 限制了行尾序号。行尾序号其实指的是换行符，在windows 中的换行符是 '\r\n'，在 linux 中的换行符是 '\n'

    WindowsLinux/MacOld Mac(pre-OSX)
    CRLFLFCR
    '\n\r''\n''\r'

因为 prettier 的默认值是 LF，但是在 windows 里面使用的是 CRLF，所以就会出现一个 Delete 'CR' 的东西了，解决的方案就是关闭这个限制。

// 新建一个 .prettierrc.js 文件
module.exports = {
  endOfLine: 'auto', //结束行形式设置成 auto 也就是不限制
}
  • 第三个爆红的地方是在一些语法格式上，句子的后面需要加 ; 或者说只能使用 ""，其实这些都是 eslint 的一些规则限制。

拓展多平台处理 #

有一个比较大的问题就是如果是基于多平台的话，假如当前展示的是知乎的，现在想切到微博上，按照代码中的写法 components 的每个组件的数据获取都是在组件内部的，但是现在平台切换了，我们怎么去通知对应的组件去更新数据呢？有一种解决方法就是通过 ref 的方式，详见 home.vue 以 wordcloud 为例。

// WorcloudChart.vue
const option = { } // 放到外面

export default {

  methods: {// option api 的写法
    getData(param) { // 根据param去请求接口
      // 获取数据 -- 提供给父组件进行调用从而更新内部的数据
      // 本质上是对图里面的 wordData 数据进行更新
      // 将 myChart2 定义在 data 这里才能进行引用
      // 之前放在 initCharts 的作用域里面这里是没办法拿到的
      axios.get('/wordcloud', {
        method: 'GET'
      }).then(res => {
        console.log(res.data)
        option.series[0].data = res.data.data
        this.myChart2.setOption(option)
      })
    },

    initCharts() {
      // console.log(this.worddata);
      let a = this.$refs.chart;

      // 换成 this.myChart2 的方式去赋值
      this.myChart2 = this.$echarts.init(a);

      this.myChart2.setOption(option);
      
      window.addEventListener('resize', () => {
        this.myChart2.resize()
      })
    },
  },
};

// Home.vue
  • 代码中将 option 放到外面去了，因为 option 里面形如背景或者颜色的一些设置其实都是公用的，也就是不受数据的影响，所以其实可以充当一个全局变量。
  • 代码中将 initCharts 中的 myChart2 写成 data 里面的变量，因为只有这样才能在触发 getData 的时候实现重载 myChart2 的 option 配置。

总结 #

eslint 和 prettier 的一些东西其实不需要说特别地去学习，有空的时候去他们的官网找下自己喜欢的代码规则，然后配置一下，后面所有的项目都可以使用，其实我自己开发的时候也没有特别注意这些代码格式的东西。然后就是可以去想一下怎么去从当前的静态拓展到动态，因为可能动态页面相比于静态的结构变化还是很多的。还有一个就是 option api 和 composition api 这次做的其他的就好像没啥问题了，实际遇到问题再实际解决就好。
