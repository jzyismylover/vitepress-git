代码随想录 #

这三个网站我在复习的过程中都有涉猎过，总的来说，我觉得他们之间的其实是有层次关系的。代码随想录里面分了很多专题，从字符串 -> 数组 -> .....，难度也是慢慢地往后走的过程，所以总的来说对于刚开始刷算法还在不太熟悉的过程推荐先按照这里面的目录先去熟悉一遍，因为它里面的题不会说特别难，基本上都是简单+中等题，对于我们思维的提高还是非常有帮助的。

在刷的时候希望你能多多思考一些，做完多问自己一些问题，开始的时候虽然说不需要太拓展地去学什么高深的解法，但是可以尝试下去更换下题意看下自己有没有办法解决。

/* 给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target ，
写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。*/

var search = function(nums, target) {
    const length = nums.length
    let left = 0, right = length - 1
    let mid = 0

    while(left <= right) {
        mid = Math.floor((left + right) / 2) // ??? 会不会溢出
        if(nums[mid] === target) {
            return mid
        } else if (nums[mid] < target) {// 向右查找
            left = mid + 1
        } else { // 向左查找
            right = mid - 1
        }
    }

    return -1
}

举个例子，就像上面这道题目一样，题目要求我们找 nums 中是否存在等于 target 的元素返回下标，那我们可以拓展一下去找 大于 target 的最小元素下标或者 小于target 最小元素下标，其实本质上是在上面的代码进行边界条件的修改，这样子可能最后学习到的内容就会很多。

相对于代码随想录来说有些专题的难度会比较大，因为它不单纯是将基础算法的网站，它也会拓展地去讲一些算法的应用什么的，但是有些模块它的解题思路真的很好，也给了一系列代码的模版，但是一开始去看的我觉得上手的难度会有一些大。

吴师兄学算法 #

这里面的题目全部都是剑指 offer 系列里面的题目，也就是企业常出的一些算法题，这个其实可以等到2023年的寒假再去刷，刷的时候就可以去查漏补缺什么的。

总结 #

先从代码随想录的专题开始刷吧！！！

总的来说，虽然算法对于我们前端来说的要求不算很高，但是不提前做准备的话还是会吃亏的。在真正的笔试中考的比较多的是数组、字符、栈与队列、贪心这些系列，其他也不是说不考但是可能会比较少一些，但有时间的话还是建议去看一下。刷的时候就直接用 javascript 去刷啦，因为这样除了说学到一些算法的知识外还可以帮你巩固 JS 的基础知识，比如数组上都有哪些方法，或者对象上有哪些方法等等。

完事开头难，一天一两道就差不多了，一年下来也有200 300道了，时间尽量控制在1-2个小时以内，因为过长的时间其实也是不划算的，毕竟这个其实也不是我们的主项，可以花更多的时间去学习前端上面的知识。坚持坚持坚持就是胜利啦！！！
